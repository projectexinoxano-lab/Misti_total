Informe de vulnerabilitats i conflictes (focalitzat en el prompt)
1. Dependències Web no necessàries
* Situació actual:
 El projecte existent conté dependències pròpies de la plataforma web (react-dom, react-native-web, @expo/metro-runtime).
Aquestes generen conflictes en execució mòbil i obliguen a versions que no sempre són compatibles amb SDKs recents.

* Risc:

   * Major pes del bundle.

   * Possibles errors de Metro bundler quan s’executa només en Android/iOS.

   * Instal·lació forçada de versions no suportades per React Native.

      * Solució al prompt:
 El nou projecte exclourà "web" de platforms i no instal·larà cap dependència web.

________________


2. Conflictes de versions React / React Native
         * Situació actual:

            * Versions desalineades entre React, React Native i Expo SDK.

            * El teu projecte fa servir react-dom@18.2.0 i altres dependències que no quadren amb React 19 i RN 0.79.x (que són el core de SDK 53).

               * Risc:

                  * Errors en temps de compilació (Invariant Violation, React is not defined, etc.).

                  * Llibreries Expo poden fallar per canvis en l’API interna de React.

                     * Solució al prompt:
 npx create-expo-app amb SDK 53 estableix versions exactes compatibles i el prompt fixa aquestes versions sense rangs (^ o ~).

________________

                                 * Dificultat per reproduir errors.

 El package.json final mostrarà versions exactes (pinned), i l’instal·lació serà controlada.
